# Eric Chapman | Writings & Speaking Archive

A durable, Markdown-first repository for long-form writing and conference content.

## Executive Summary

This repository is the canonical source for Eric Chapman’s published thought leadership across engineering strategy, platform modernization, quality transformation, and enterprise AI adoption.

It is designed for long-term maintainability, clean reuse across channels, and straightforward future publishing to web, newsletter, and syndication platforms.

## Audience

- Engineering and technology executives
- Platform engineering and DevOps leaders
- Quality engineering and transformation teams
- Enterprise architects and delivery organizations

## Content Domains

| Domain | Description |
|---|---|
| Enterprise AI | Practical adoption patterns, governance, and delivery realities |
| Developer Experience | Delivery flow, platform usability, and team productivity |
| Engineering Leadership | Organizational design, enablement models, and transformation strategy |
| Platform & DevOps | CI/CD systems, Kubernetes operations, and platform capabilities |
| Quality Engineering | Built-in quality, continuous testing, and defect prevention |
| Enterprise Transformation | Operating model evolution and modernization execution |

## Repository Layout

| Path | Purpose |
|---|---|
| `content/writing/` | Canonical article source, organized by primary category |
| `content/speaking/` | Speaker profile and engagement portfolio content |
| `data/article-index.json` | Machine-readable article registry |
| `data/tags.json` | Normalized tag catalog with usage counts |
| `data/redirects.json` | Canonical writing route mappings |
| `data/speakers.json` | Structured speaker and session metadata |
| `docs/CONTENT_MODEL.md` | Content model and schema decisions |
| `docs/NEXT_STEPS.md` | Future roadmap (not yet implemented) |
| `scripts/` | Validation, generation, and normalization tooling |
